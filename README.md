# NovelForge v2.3.1

> 长篇小说全自动写作系统 — 代码驱动 + AI 监控

NovelForge 是一个代码驱动的长篇小说自动写作系统。Python 脚本负责流程执行（写作循环、审核、审计、记忆更新），OpenClaw AI 负责监控和兜底。两者结合，代码保证步骤不被偷省，AI 保证决策不出错。

## 核心特性

### 写作方法论（融合 InkOS）

- **25 条核心规则**：人物塑造铁律、叙事技法、逻辑自洽、语言约束
- **六步走心理分析**：当前处境→核心动机→信息边界→性格过滤→行为选择→情绪外化
- **创作宪法 14 条**：Show don't tell、盐溶于汤、全员智商在线...
- **代入感六支柱**：标签化、熟悉感、共鸣、欲望、五感、人设活化
- **黄金三章纪律**：开篇硬约束，决定读者是否追读
- **信息边界铁律**：角色只能基于已知信息行动，防止信息越界
- **反例→正例对照表**：具体的去 AI 味修改示例

### 质量保障体系

- **AI 痕迹 5 维度检测**：段落等长、套话密度、公式化转折、列表结构、高疲劳词
- **硬规则 Spot-fix**：破折号、违禁句式、分析术语、章节号指称自动修正
- **18 维度审计**：3 层分级（每章/每 5 章/每 20 章），覆盖角色名、时间线、资源、信息越界、伏笔回收、支线停滞、节奏张弛等
- **审计→修订循环**：warning 级审计问题自动触发修订
- **内容审核**：衔接、矛盾、人设、大纲、信息边界、违禁项

### 记忆系统

| 文件 | 用途 |
|------|------|
| 全局记忆.docx | 世界观核心规则、角色档案 |
| 线索台账.docx | 角色状态、物品道具、伏笔、时间线 |
| 角色信息边界.docx | 每个角色知道什么、不知道什么 |
| 资源账本.docx | 物品归属、数量、状态追踪 |
| 情感弧线.docx | 角色情绪变化追踪 |
| 近期摘要.docx | 最近 8 章摘要（滑动窗口） |
| 章节大纲索引.docx | 按章拆分的大纲 |
| 写作进度.docx | 断点续写状态 |

### 流程保障

- **材料确认**：启动前 AI 检查材料完整性、逻辑矛盾
- **精简上下文**：按相关性筛选台账条目，避免 token 爆炸
- **4 轮异常处理**：原始重试→精简重试→冷却重试→暂停
- **截断检测**：输出被 token 限制截断时自动重写
- **阶段审核**：每 50 章触发台账审查
- **终审**：全书完成后 3 轮终审
- **断点续写**：中断后从断点继续
- **崩溃保护**：原子写入（先写 .tmp 再替换），防止文件损坏

## 快速开始

### 安装依赖

```bash
pip install openai python-docx
```

### 方式一：独立运行（不使用 OpenClaw）

```bash
python novelforge_v2.3.1.py
```

按提示输入 API 配置、项目信息、故事圣经和章节大纲即可。

### 方式二：配合 OpenClaw 使用（推荐）

1. 将文件放入 OpenClaw skill 目录：

```
~/.openclaw/skills/novelforge-writer/
├── SKILL.md                  ← AI 读取的指令文件
└── novelforge_v2.3.1.py      ← 写作脚本
```

2. 在 OpenClaw 中发送"开始写"，AI 会自动收集材料、配置脚本、启动运行并持续监控。

3. 脚本支持命令行参数模式，AI 通过以下命令启动脚本：

```bash
python novelforge_v2.3.1.py --auto \
  --base-url "API地址" \
  --api-key "API密钥" \
  --model "模型名" \
  --name "小说名" \
  --chapters 500 \
  --words 2500 \
  --bible "故事圣经文件路径" \
  --outlines "章节大纲文件路径" \
  --mode new
```

## 配置项

在脚本运行时交互输入，或通过命令行参数指定：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| --name | 我的小说 | 项目名称，决定输出文件夹名 |
| --chapters | 500 | 总章节数 |
| --words | 2500 | 每章目标字数（±300） |
| --base-url | - | API Base URL |
| --api-key | - | API Key |
| --model | - | 模型名称 |
| --bible | - | 故事圣经文件路径 |
| --outlines | - | 章节大纲文件路径 |
| --mode | new | 运行模式：new / resume / import |
| --start | 1 | 导入模式的起始章节号 |
| --folder | - | 导入模式的已完成章节文件夹路径 |
| --style | - | 写作风格要求 |
| --special | - | 其他特殊要求 |
| --review | - | 自定义审核标准 |

内部配置项（在代码 `Config` 类中）：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| recent_summary_window | 8 | 近期摘要保留章节数 |
| batch_review_interval | 50 | 阶段审核间隔（每 N 章触发） |
| chapter_retry_max | 3 | 单章审核最大重试次数 |
| final_review_max_rounds | 3 | 终审最大轮数 |
| max_audit_revise_attempts | 2 | 审计修订最大次数 |
| transition_word_interval | 1000 | 转折词密度阈值（每 N 字允许 1 次） |

# NovelForge

> 长篇小说全自动写作系统

NovelForge 是一个 AI 驱动的长篇小说自动写作系统。它负责调度、审核、记忆管理和保存，AI 子进程只负责写正文。覆盖从材料确认到终审的完整写作流程。

## 核心特性

### 写作方法论（融合 InkOS）

- **25 条核心规则**：人物塑造铁律、叙事技法、逻辑自洽、语言约束
- **六步走心理分析**：当前处境→核心动机→信息边界→性格过滤→行为选择→情绪外化
- **创作宪法 14 条**：Show don't tell、盐溶于汤、全员智商在线...
- **代入感六支柱**：标签化、熟悉感、共鸣、欲望、五感、人设活化
- **黄金三章纪律**：开篇硬约束，决定读者是否追读
- **信息边界铁律**：角色只能基于已知信息行动，防止信息越界
- **反例→正例对照表**：具体的去 AI 味修改示例

### 质量保障体系

- **AI 痕迹 5 维度检测**：段落等长、套话密度、公式化转折、列表结构、高疲劳词
- **硬规则 Spot-fix**：破折号、违禁句式、分析术语、章节号指称自动修正
- **18 维度审计**：3 层分级（每章/每 5 章/每 20 章），覆盖角色名、时间线、资源、信息越界、伏笔回收、支线停滞、节奏张弛等
- **审计→修订循环**：warning 级审计问题自动触发修订
- **内容审核**：衔接、矛盾、人设、大纲、信息边界、违禁项

### 记忆系统

| 文件 | 用途 |
|------|------|
| 全局记忆.md | 世界观核心规则、角色档案 |
| 线索台账.md | 角色状态、物品道具、伏笔、时间线 |
| 角色信息边界.md | 每个角色知道什么、不知道什么 |
| 资源账本.md | 物品归属、数量、状态追踪 |
| 情感弧线.md | 角色情绪变化追踪 |
| 近期摘要.md | 最近 3 章摘要（滑动窗口） |
| 章节大纲索引.md | 按章拆分的大纲 |
| 写作进度.md | 断点续写状态 |

### 流程保障

- **材料确认**：启动前 AI 检查材料完整性、逻辑矛盾
- **精简上下文**：按相关性筛选台账条目，避免 token 爆炸
- **4 轮异常处理**：原始重试→精简重试→冷却重试→暂停
- **截断检测**：输出被 token 限制截断时自动重写
- **阶段审核**：每 50 章触发台账审查
- **终审**：全书完成后 3 轮终审
- **断点续写**：中断后从断点继续

## 两种使用方式

### 方式一：Python 版（推荐批量跑）

```bash
pip install python-docx openai
python novelforge.py
```

按提示输入 API 配置、项目信息、材料即可自动运行。

所有检测、审计、spot-fix 由代码执行，100% 稳定。

### 方式二：Skill 版（推荐交互式创作）

将 `SKILL.md` 作为 OpenClaw Skill 安装，或直接粘贴到 Agent 对话中。

```
clawhub install novel-forge
```

检测、审计由 Agent（LLM）执行，部署零依赖。

### 对比

| | Python 版 | Skill 版 |
|---|---|---|
| 检测/审计 | 代码执行，100% 稳定 | Agent 执行，~90% 稳定 |
| 信息越界检测 | 代码，有误报 | Agent 判断，更准 |
| 崩溃保护 | atomic_write | 无 |
| 部署 | 需要 Python 环境 | 装上就能用 |
| 适用场景 | 批量跑 500 章 | 交互式创作 |

## 工作流程

```
材料确认
  ↓
初始化（生成记忆系统）
  ↓
逐章循环 ─────────────────────────┐
  ├── 构建精简上下文               │
  ├── 主进程决策                   │
  ├── 子进程写作                   │
  ├── Spot-fix（硬规则修正）        │
  ├── AI 痕迹检测 + 修订           │
  ├── 内容审核 + 重写              │
  ├── 18 维度审计 + 修订           │
  ├── 保存章节                     │
  ├── 更新记忆系统                 │
  └── 每 50 章触发阶段审核 ────────┘
  ↓
终审（最多 3 轮）
  ↓
完成
```

## 快速开始

### 准备材料

1. **故事圣经**：世界观设定、核心角色档案（姓名/年龄/身份/性格/关键特征/初始状态）、主线/支线剧情框架
2. **章节大纲**：每一章的剧情要点，越详细越好
3. **总章节数**和**每章目标字数**

### 运行

```bash
# Python 版
python novelforge.py

# 选择模式
# 1. 开始写（全新写作）
# 2. 继续写（断点续写）
# 3. 从第X章开始续写（导入模式）
```

### 配置

```python
# Python 版通过交互式输入配置
config.ai_base_url = "https://api.example.com/v1"
config.ai_api_key = "sk-..."
config.ai_model = "model-name"
config.project_name = "吞天魔帝"
config.total_chapters = 500
config.words_per_chapter = 2500
```
